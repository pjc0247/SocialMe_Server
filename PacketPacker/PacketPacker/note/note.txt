
DB
	한개의 유저 -> 한개의 db connection

분산 처리
	메세지 수신 -> 대상이 자기 서버에 있나 검사 ->
	있으면 -> 대상에 메세지 전송
	없으면 -> 다른 서버에 메세지와 대상 전송 -> 해당 서버에서 존재하면 해당 서버에서 메세지 전송

유저 정보 저장
	로그인시 -> 유저 map에 유저id를 key로 삼고 접속 정보를 박는다
	로그아웃시 -> 유저 map에서 뺸다

	메세지 수신 -> map에서 대상 id를 key로 접속 정보를 찾아낸다 -> 대상에 메세지 전송


비정상 연결 종료 처리
	핑을 통해 매 시간마다 살았는지 체크할 것인지,
	마지막 데이터 수신으로부터 일정 시간이 지나면 죽일 것인지
